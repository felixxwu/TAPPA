How should the catalog scene render the curves?

The definition is a 2D Curve2D. The catalog scene lays each corner out in a row so you can eyeball the shapes. Three ways to draw them:

A · Pure 2D (Line2D)

A Node2D scene, each curve drawn as a flat Line2D. Truest to the 2D definition, zero 3D setup, fastest to build.

B · Road ribbon (3D mesh)

Lay each curve on a flat ground plane (X-Z), extrude a constant-width road mesh with the PS1 flat shader + dashed centerline. This mesh is the actual track primitive you'll reuse for generation.

C · Centerline + control handles

Thin tube/line on flat ground, plus visible bezier control points & tangent handles. Debug-focused — great for authoring/tuning curves, less like the real track.

My recommendation: B, with C's dashed centerline drawn on top. The ribbon is the primitive you'll actually reuse for track generation, flat-shaded to match the PS1 look, and a top-down orthographic camera makes the shapes easy to compare. Each corner gets a text label ("Right 4", "Hairpin", etc.).
